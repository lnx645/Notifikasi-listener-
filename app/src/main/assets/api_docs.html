Dokumentasi API Bridge

Integrasikan Notifikasi Android dengan Server Mandiri Anda

Pengenalan

Notification Bridge adalah aplikasi Android yang berfungsi untuk meneruskan (bridge) setiap notifikasi masuk di perangkat Android ke server backend kustom Anda secara real-time.

Aplikasi ini sangat cocok digunakan untuk otomasi sistem, integrasi chatbot, pencatatan transaksi otomatis (seperti mutasi e-wallet), maupun sinkronisasi notifikasi antar perangkat.

Skema Keamanan (Auth Token)

Jika Anda mengaktifkan pengaturan Auth Token di menu Settings aplikasi, setiap request HTTP yang dikirimkan oleh aplikasi ke server Anda akan menyertakan Header otorisasi standar:

Authorization: Bearer <Token_Anda>

Server Anda disarankan untuk memeriksa header ini guna memastikan bahwa data yang masuk benar-benar berasal dari perangkat terpercaya Anda.

POST Kirim Notifikasi

Dipanggil secara otomatis oleh aplikasi setiap kali mendeteksi ada notifikasi masuk baru di perangkat Anda (sesuai filter).

POST [BASE_URL]/notification

Header Request:

  • Content-Type: application/json
  • User-Agent: NotificationBridge/1.0
  • Authorization: Bearer [TOKEN] (jika diaktifkan)

Payload Request (JSON):

Parameter Tipe Keterangan
device_id String ID Unik Perangkat (didapatkan dari serial/Android ID).
device_name String Nama Perangkat yang disetel di menu settings.
package_name String Package ID aplikasi pengirim (cth: com.whatsapp).
application_name String Nama Aplikasi pengirim (cth: WhatsApp).
title String Judul notifikasi yang tampil di bar notifikasi.
message String Isi pesan atau tubuh notifikasi.
big_text String? Teks panjang opsional jika notifikasi memiliki preview panjang.
sub_text String? Sub-teks opsional dari bar notifikasi.
received_at String Waktu intercept lokal (Format ISO 8601).
android_version String Versi SDK Android perangkat pengirim.
device_brand String Merk manufaktur handphone (cth: Samsung).
device_model String Model spesifik perangkat (cth: SM-G991B).

Contoh JSON Body:

{
  "device_id": "89ab-cd34-ef56",
  "device_name": "HP Utama Dadan",
  "package_name": "com.whatsapp",
  "application_name": "WhatsApp",
  "title": "Budi",
  "message": "Halo, jangan lupa kirim berkas integrasi API ya!",
  "big_text": null,
  "sub_text": null,
  "received_at": "2026-07-01T18:58:00Z",
  "android_version": "14",
  "device_brand": "Google",
  "device_model": "Pixel 8 Pro"
}

Response yang Diharapkan:

Server Anda harus mengembalikan HTTP Status Code 200 OK (atau kode sukses 2xx lainnya) dengan cepat untuk menandai bahwa data berhasil diterima dengan baik.

GET Sinkronisasi Package

Digunakan oleh aplikasi untuk mengambil daftar package name aplikasi mana saja yang ingin disaring (di-intercept). Jika endpoint ini mengembalikan daftar kosong, aplikasi akan menyaring semua notifikasi yang diizinkan oleh sistem.

GET [BASE_URL]/config/packages

Header Request:

  • User-Agent: NotificationBridge/1.0
  • Authorization: Bearer [TOKEN] (jika diaktifkan)

Response Body (JSON) yang Wajib Dikembalikan:

{
  "packages": [
    "com.whatsapp",
    "id.dana",
    "com.shopee.id",
    "com.google.android.gm"
  ]
}

Contoh Kode Backend (PHP)

Berikut adalah contoh script PHP sederhana (index.php) untuk menerima notifikasi dan menyimpannya ke dalam file log di server Anda:

<?php
// Ambil header otorisasi
$headers = getallheaders();
$authHeader = $headers['Authorization'] ?? '';

// Verifikasi Token (Jika token diatur ke 'SANGATRAHASIA')
if ($authHeader !== 'Bearer SANGATRAHASIA') {
    http_response_code(401);
    echo json_encode(["error" => "Unauthorized Access!"]);
    exit;
}

// Ambil input JSON dari aplikasi
$inputJSON = file_get_contents('php://input');
$data = json_decode($inputJSON, true);

if ($data) {
    // Proses data (Contoh: simpan ke database atau kirim ke bot Telegram)
    $logText = "[" . $data['received_at'] . "] " . $data['application_name'] . " - " . $data['title'] . ": " . $data['message'] . PHP_EOL;
    file_put_contents('notif_bridge.log', $logText, FILE_APPEND);

    // Balas dengan status sukses
    http_response_code(200);
    echo json_encode(["status" => "success", "message" => "Notification logged."]);
} else {
    http_response_code(400);
    echo json_encode(["error" => "Invalid Payload"]);
}
